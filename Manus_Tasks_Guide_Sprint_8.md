# دليل تنفيذ مهام Sprint 8 (النشر والتشغيل)

هذا الدليل يحتوي على البرومبتات الجاهزة لتنفيذ مهام السبرنت الثامن والأخير (Sprint 8) الخاص بنشر منصة Tendex AI على خادم Hostinger VPS.

## ملاحظات هامة قبل البدء:
- يجب تنفيذ المهام **بالترتيب** (801 ثم 802 ثم 803).
- ستحتاج إلى تزويد الوكيل ببيانات الوصول للخادم (IP وكلمة مرور Root) عند طلبها.
- تأكد من توجيه الدومين (DNS) إلى IP الخادم قبل البدء في TASK-802.

---

## TASK-801: الإعداد الأولي للخادم وبيئة الإنتاج (Initial Server Setup)

**البرومبت:**
```markdown
# السياق والهوية (Context & Role)
أنت مهندس عمليات (DevOps Engineer) تعمل ضمن فريق بناء منصة **Tendex AI**. دورك الحالي هو إعداد خادم Hostinger VPS الجديد (Ubuntu 24.04 LTS) وتجهيز بيئة الإنتاج.

# التعليمات الصارمة (Strict Instructions)
- **الأمان:** يُمنع منعاً باتاً تخزين كلمات المرور أو مفاتيح SSH في الكود المصدري أو رفعها إلى GitHub.
- **الاعتمادية:** استخدم السكربتات الموجودة مسبقاً في مجلد `infrastructure/scripts/` ولا تقم بإنشاء سكربتات جديدة إلا للضرورة القصوى.
- **التوثيق:** سجل جميع الخطوات التي قمت بها بدقة.

# الوثائق المطلوبة للمهمة (Required Reading)
1. `PROGRESS.md`
2. `docs/Deployment_Guide.md`
3. `infrastructure/scripts/server-setup.sh`

# المطلوب تنفيذه (The Task)
- **رقم المهمة:** TASK-801
- **عنوان المهمة:** الإعداد الأولي للخادم وبيئة الإنتاج
- **الوصف:**
  1. اطلب من المستخدم تزويدك بـ IP الخادم وكلمة مرور Root.
  2. اتصل بالخادم عبر SSH.
  3. قم بنقل مجلد `infrastructure` إلى الخادم في المسار `/opt/tendex-ai/infrastructure`.
  4. قم بتشغيل سكربت `server-setup.sh` على الخادم لإعداد Docker وجدار الحماية.
  5. قم بإنشاء ملف `.env.prod` على الخادم بناءً على `infrastructure/.env.prod.example` وقم بتوليد كلمات مرور قوية وحقيقية لجميع الخدمات (SQL Server, Redis, RabbitMQ, MinIO, Qdrant, JWT, إلخ).
  6. اطلب من المستخدم إضافة أسرار GitHub (GitHub Secrets) التالية في مستودع المشروع: `VPS_HOST`, `VPS_USER`, `VPS_SSH_KEY`, `VPS_PORT`. (قم بتوليد مفتاح SSH جديد على الخادم وأضف المفتاح العام إلى `authorized_keys` وأعطِ المفتاح الخاص للمستخدم ليضيفه في GitHub).
  7. قم بإعادة تشغيل الخادم (reboot) لتطبيق التغييرات.

- **معايير القبول (Acceptance Criteria):**
  - الخادم معد بالكامل ويحتوي على Docker و Docker Compose.
  - جدار الحماية (UFW) مفعل ويسمح بمنافذ 22, 80, 443 فقط.
  - ملف `.env.prod` موجود على الخادم ويحتوي على بيانات حقيقية.
  - مفتاح SSH جاهز للاستخدام في GitHub Actions.

**خطوات التسليم:**
1. قم بتنفيذ الخطوات على الخادم.
2. قم بتحديث ملف `PROGRESS.md` ليعكس إنجاز هذه المهمة.
3. قم برفع التغييرات (إن وجدت) إلى GitHub.
```

## TASK-802: إعداد SSL والنشر الأول (SSL & First Deployment)

**البرومبت:**
```markdown
# السياق والهوية (Context & Role)
أنت مهندس عمليات (DevOps Engineer) تعمل ضمن فريق بناء منصة **Tendex AI**. دورك الحالي هو إعداد شهادات SSL وتنفيذ أول عملية نشر للمنصة على خادم الإنتاج.

# التعليمات الصارمة (Strict Instructions)
- تأكد من أن المستخدم قد قام بتوجيه الدومين (DNS) إلى IP الخادم قبل تشغيل سكربت SSL.
- راقب سجلات النشر (Logs) للتأكد من عدم وجود أخطاء أثناء بدء تشغيل الحاويات.

# الوثائق المطلوبة للمهمة (Required Reading)
1. `PROGRESS.md`
2. `docs/Deployment_Guide.md`
3. `infrastructure/scripts/init-ssl.sh`
4. `infrastructure/scripts/deploy.sh`

# المطلوب تنفيذه (The Task)
- **رقم المهمة:** TASK-802
- **عنوان المهمة:** إعداد SSL والنشر الأول
- **الوصف:**
  1. اطلب من المستخدم تأكيد الدومين (مثال: tendex.sa) والبريد الإلكتروني، وتأكيد توجيه DNS.
  2. اتصل بالخادم عبر SSH.
  3. قم بتشغيل سكربت `init-ssl.sh` لإصدار شهادات Let's Encrypt للدومين المحدد.
  4. قم بتشغيل مسار النشر التلقائي (CD Pipeline) من GitHub Actions، أو قم بتشغيل سكربت `deploy.sh up` يدوياً على الخادم لبدء جميع الخدمات.
  5. راقب حالة الحاويات باستخدام `docker ps` وتأكد من أن جميع الخدمات (Backend, Frontend, SQL Server, Redis, إلخ) في حالة `healthy`.

- **معايير القبول (Acceptance Criteria):**
  - شهادة SSL صالحة ومثبتة بنجاح.
  - جميع حاويات Docker تعمل وحالتها `healthy`.
  - يمكن الوصول إلى الواجهة الأمامية عبر `https://[domain]`.
  - يمكن الوصول إلى الواجهة الخلفية عبر `https://[domain]/api/health`.

**خطوات التسليم:**
1. قم بتنفيذ الخطوات على الخادم.
2. قم بتحديث ملف `PROGRESS.md` ليعكس إنجاز هذه المهمة.
3. قم برفع التغييرات (إن وجدت) إلى GitHub.
```

## TASK-803: التحقق النهائي وإعداد مدير النظام (Verification & Admin Setup)

**البرومبت:**
```markdown
# السياق والهوية (Context & Role)
أنت مهندس جودة وعمليات (QA & DevOps Engineer) تعمل ضمن فريق بناء منصة **Tendex AI**. دورك الحالي هو التحقق من صحة النشر وإعداد حساب المشغل الأساسي (Super Admin).

# التعليمات الصارمة (Strict Instructions)
- لا تقم بتعديل الكود المصدري إلا إذا اكتشفت خطأ حرجاً يمنع تشغيل المنصة.
- تأكد من عمل جميع التكاملات (قاعدة البيانات، التخزين، طابور الرسائل).

# الوثائق المطلوبة للمهمة (Required Reading)
1. `PROGRESS.md`
2. `docs/Deployment_Guide.md`

# المطلوب تنفيذه (The Task)
- **رقم المهمة:** TASK-803
- **عنوان المهمة:** التحقق النهائي وإعداد مدير النظام
- **الوصف:**
  1. اتصل بالخادم عبر SSH.
  2. قم بتشغيل سكربت `deploy.sh health` للتحقق من صحة جميع الخدمات.
  3. قم بالدخول إلى حاوية قاعدة البيانات (SQL Server) وتأكد من تطبيق جميع الـ Migrations بنجاح على قاعدة بيانات `master_platform`.
  4. قم بإنشاء حساب المشغل الأساسي (Super Admin) مباشرة في قاعدة البيانات (أو عبر API مخصص إن وجد) باستخدام بريد إلكتروني وكلمة مرور يحددها المستخدم.
  5. قم بإعداد مهام Cron للنسخ الاحتياطي التلقائي (`backup.sh`) والمراقبة (`monitor.sh`) كما هو موضح في `Deployment_Guide.md`.
  6. قم بإجراء اختبار دخول سريع (Login) من خلال الواجهة الأمامية للتأكد من عمل النظام بالكامل.

- **معايير القبول (Acceptance Criteria):**
  - جميع الخدمات تعمل بكفاءة وبدون أخطاء في السجلات (Logs).
  - مهام Cron للنسخ الاحتياطي والمراقبة مجدولة وتعمل.
  - حساب Super Admin جاهز ويمكن تسجيل الدخول به.
  - المنصة جاهزة تماماً للاستخدام الفعلي.

**خطوات التسليم:**
1. قم بتنفيذ الخطوات على الخادم.
2. قم بتحديث ملف `PROGRESS.md` ليعكس إنجاز هذه المهمة واكتمال المشروع.
3. قم برفع التغييرات (إن وجدت) إلى GitHub.
```
