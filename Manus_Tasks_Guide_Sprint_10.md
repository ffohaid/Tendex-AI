### 2.1 الواجهة الأمامية (Frontend)
- بناء مكون Drag & Drop تفاعلي لتصميم مسارات الاعتماد باستخدام مكتبة مثل vue-flow أو @vue-flow/core
- أنواع العقد (Nodes):
  * عقدة البداية (Start)
  * عقدة الموافقة (Approval) - تحتوي على: المعتمد، SLA، إجراء التصعيد
  * عقدة التفرع المشروط (Conditional Branch) - شرط if/else
  * عقدة التنفيذ المتوازي (Parallel) - مسارات متعددة تعمل بالتوازي
  * عقدة النهاية (End)
- أنواع المسارات:
  * تسلسلي (Sequential): A → B → C
  * متوازي (Parallel): A → [B, C] → D
  * مشروط (Conditional): A → if(condition) → B else → C
- خصائص كل عقدة:
  * اسم المرحلة (عربي + إنجليزي)
  * المعتمد (مستخدم / دور / لجنة)
  * مدة SLA (بالساعات/الأيام)
  * إجراء التصعيد عند تجاوز SLA
  * إجراءات متاحة (موافقة، رفض، إعادة، تعليق)
- شريط أدوات جانبي لسحب العقد
- خطوط ربط (Edges) قابلة للتخصيص
- أزرار: حفظ، حفظ كمسودة، معاينة، تفعيل
- عرض مصغر (Minimap) للمسارات الكبيرة

### 2.2 الواجهة الخلفية (Backend)
- إنشاء Entities: WorkflowTemplate, WorkflowNode, WorkflowEdge, WorkflowInstance, WorkflowStepInstance
- APIs:
  * POST /api/v1/workflows - إنشاء مسار جديد
  * GET /api/v1/workflows - قائمة المسارات
  * GET /api/v1/workflows/{id} - تفاصيل مسار
  * PUT /api/v1/workflows/{id} - تعديل مسار
  * POST /api/v1/workflows/{id}/activate - تفعيل مسار
  * POST /api/v1/workflows/{id}/instances - بدء تنفيذ مسار
  * POST /api/v1/workflows/instances/{id}/steps/{stepId}/action - تنفيذ إجراء على خطوة
- محرك سير العمل (Workflow Engine):
  * تنفيذ المسارات التسلسلية والمتوازية والمشروطة
  * مراقبة SLA وإطلاق التصعيد التلقائي (تنبيه مبكر → تصعيد أول → تصعيد ثاني)
  * تسجيل كل إجراء في سجل التدقيق (Audit Trail)
  * إرسال إشعارات عند كل خطوة

### 14.1 جودة الكود
- جميع أسماء المتغيرات والجداول والتعليقات باللغة الإنجليزية
- لا بيانات وهمية (Mock Data) - كل شيء من APIs حقيقية

- التحقق من المدخلات في الفرونت (VeeValidate + Zod) والباك (FluentValidation)
- معالجة أخطاء شاملة (Error Handling) في كل صفحة وكل API

### 14.2 الأمان
- Access Token: 60 دقيقة، Refresh Token: 8 ساعات
- سجل تدقيق غير قابل للتعديل (Immutable Audit Trail)
- تشفير البيانات الحساسة
- حماية من SQL Injection (Parameterized Queries)
- حماية من XSS وCSRF

### 14.3 الأداء
- زمن تحميل الصفحات أقل من 1.5 ثانية
- زمن استجابة APIs أقل من 300 مللي ثانية
- Lazy Loading للمكونات الثقيلة
- تخزين مؤقت (Redis) للبيانات الثابتة
