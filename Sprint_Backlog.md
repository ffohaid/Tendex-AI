# خطة التنفيذ المفصلة (Sprint Backlog) لمنصة Tendex AI

تم إعداد هذه الخطة الشاملة بناءً على وثائق المشروع المرجعية (PRD v6، Architecture v1، Execution Plan، وغيرها) لتقسيم تطوير منصة Tendex AI إلى مهام تطويرية صغيرة ومستقلة (Micro-tasks) موزعة على 7 سبرنتات (Sprints). تهدف هذه الخطة إلى توجيه وكلاء التطوير لضمان بناء نظام متكامل، آمن، وقابل للتوسع.

## القواعد العامة للتطوير

يجب على جميع وكلاء التطوير الالتزام الصارم بالقواعد التالية أثناء تنفيذ أي مهمة:

- **التقنيات المعتمدة:** يُسمح فقط باستخدام التقنيات المحددة وهي Vue.js 3 (Composition API), TypeScript, Tailwind CSS v4, PrimeVue 4, Pinia 2 للواجهة الأمامية. وللواجهة الخلفية .NET 10 LTS (C# 14), Clean Architecture, CQRS (MediatR), Minimal APIs.
- **قواعد البيانات والتخزين:** الاعتماد على SQL Server 2022 كقاعدة بيانات علائقية، Qdrant للمتجهات، Redis 7 للتخزين المؤقت، MinIO لتخزين الملفات، و RabbitMQ للمراسلة.
- **منع البيانات الوهمية (NO MOCK DATA):** يُمنع تماماً كتابة بيانات وهمية (Hardcoded Arrays) في مكونات الواجهة. يجب جلب جميع البيانات ديناميكياً من واجهات برمجة التطبيقات (APIs).
- **جودة الكود وقواعد البيانات:** الالتزام بكتابة الكود باللغة الإنجليزية حصراً، ومنع استخدام الحذف المتسلسل (No Cascade Deletes) في قاعدة البيانات.
- **دعم اللغة والاتجاه (RTL):** المنصة ثنائية اللغة وتدعم التبديل الديناميكي. يجب استخدام الخصائص المنطقية لـ Tailwind حصرياً (مثل: ms-4, pe-2) لضمان التوافق مع اتجاه العرض.
- **التطوير الموجه بالاختبار (TDD):** يجب كتابة اختبارات الوحدة (Unit Tests) قبل أو بالتزامن مع كتابة الكود الفعلي لضمان الموثوقية.
- **التحقق الفوري (Real-time Validation):** تطبيق التحقق من المدخلات في الواجهة الأمامية باستخدام (VeeValidate + Zod) وفي الواجهة الخلفية باستخدام (FluentValidation).
- **إعدادات الذكاء الاصطناعي:** يجب تخزين مفاتيح API وإعدادات النماذج بشكل ديناميكي في جدول `AiConfiguration` مشفرة باستخدام AES-256، ويُمنع تخزينها في ملفات التكوين.
