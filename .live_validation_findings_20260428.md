## Root-cause diagnosis in progress

| Area | Evidence | Likely root cause |
| --- | --- | --- |
| Booklet list page now empty | The frontend booklet list uses `fetchRfpList()` against `/v1/competitions`. The backend competitions repository loads `Competition` entities directly from `rfp.Competitions`. A new mapped EF property `RequiredAttachmentTypesSerialized` was added and bound to a physical column named `RequiredAttachmentTypes`, while deployment does not apply any tenant-database schema change automatically. | The QC fix introduced a schema-dependent field into the tenant `Competitions` entity without a matching production migration step. If tenant databases do not yet contain `rfp.Competitions.RequiredAttachmentTypes`, all competition queries can fail at runtime, which explains the empty booklet list. |
| Supplier offers page now shows error | `SupplierOffersView.vue` loads data via `fetchCompetitionEvaluations()`, which also calls `/v1/competitions?page=1&pageSize=100` before opening any competition details. | The supplier offers page is failing for the same upstream reason as the booklet list: it depends on the competitions list endpoint, so the same competition-query failure propagates into this page. |
| Previous branding fix appears incomplete from user perspective | Production now returns a public same-origin logo URL and the logo image is downloadable with HTTP 200, but `LoginView.vue` still renders tenant branding inside a fixed `80x80` white square box. The live page visually shows a white square instead of an acceptable full tenant logo. | The backend path for logo delivery improved, but the login-page rendering path remains unsuitable for wide governmental logos, so the user still perceives the branding issue as unresolved. |
| Why this slipped through | The deployment workflow relies on backend startup for migrations and contains no explicit tenant-database schema step for this field. The earlier production validation focused mainly on login branding/network symptoms and did not include authenticated regression of competitions list and supplier-offers list after the schema-linked change. | The regression passed deployment because pipeline/build checks were green while runtime tenant schema compatibility was not validated end-to-end on production data. |
